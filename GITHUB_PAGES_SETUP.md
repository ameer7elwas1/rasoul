# إعداد GitHub Pages

## خطوات تفعيل GitHub Pages:

### 1. اذهب إلى إعدادات المستودع
- افتح المستودع على GitHub: https://github.com/ameer7elwas1/rasoul
- اضغط على **Settings** (الإعدادات)

### 2. اذهب إلى قسم Pages
- في القائمة الجانبية، اضغط على **Pages** (أو **Pages and sites**)

### 3. إعداد Source
- في قسم **Source**، اختر:
  - **Branch**: `main`
  - **Folder**: `/ (root)` أو `/رسول الرحمة` (حسب موقع الملفات)
- اضغط على **Save** (حفظ)

### 4. انتظر قليلاً
- قد يستغرق الأمر دقيقة أو دقيقتين حتى يتم نشر الموقع
- ستظهر رسالة: "Your site is live at https://ameer7elwas1.github.io/rasoul/"

## ملاحظات مهمة:

1. **ملف .nojekyll**: تم إضافته لتعطيل Jekyll (مطلوب للملفات التي تبدأ بنقطة)

2. **ملف index.html**: يجب أن يكون موجوداً في المجلد المحدد في Source

3. **الروابط**: بعد التفعيل، الموقع سيكون متاحاً على:
   - `https://ameer7elwas1.github.io/rasoul/`
   - أو `https://ameer7elwas1.github.io/rasoul/index.html`

4. **إذا كانت الملفات في مجلد فرعي**: 
   - إذا كانت الملفات في مجلد `رسول الرحمة` داخل المستودع
   - يجب تحديد `/رسول الرحمة` في Folder
   - أو نقل الملفات إلى الجذر (root)

## حل المشاكل:

### إذا ظهر خطأ 404:
1. تأكد من أن `index.html` موجود في المجلد المحدد
2. تأكد من أن Source يشير إلى المجلد الصحيح
3. انتظر بضع دقائق بعد التغيير
4. امسح الكاش (Ctrl+F5)

### إذا لم تظهر التغييرات:
- قد يستغرق نشر التغييرات من 1-5 دقائق
- امسح الكاش أو استخدم وضع التصفح الخاص
